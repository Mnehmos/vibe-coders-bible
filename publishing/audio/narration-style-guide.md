# Narration Style Guide

## Voice

The audiobook should sound like a serious field manual:

- Direct.
- Calm.
- Precise.
- Useful.
- Human-paced.

Avoid:

- Sales tone.
- Overdramatized manifesto delivery.
- Overly cheerful narration.
- Robotic monotony.

## Rhythm

Use a slightly slower pace for:

- Manifesto lines.
- Key lines.
- Definitions.
- Checklists.

Use a normal pace for:

- Explanatory prose.
- Examples.
- Case studies.

## Tables

Do not read large tables cell by cell unless the table is tiny. Convert them into spoken summaries.

Example:

```text
Instead of treating every model output the same, route each output by risk. Explanations go to human review. Code goes to tests and type checks. Shell commands, migrations, citations, and tool calls require stronger boundaries.
```

## Code blocks

For short conceptual blocks, read the structure.

For long command examples, summarize the purpose and mention that the exact command appears in the ebook and PDF.

## Repeated chapter elements

Each chapter can use this spoken structure:

1. Title.
2. Thesis.
3. Key line.
4. Main argument.
5. Practical artifact.
6. Transition.
