# Chapter 35 - The Daily Vibe-Coder Workflow

Part: VIII - The Field Manual

## Thesis

Safe AI-assisted work can be practiced as a daily loop.

## Key line

Velocity comes from repeating a small disciplined loop, not from trusting a large unreviewed leap.

## Practical loop

```text
1. Pick one issue.
2. Write acceptance criteria.
3. Ask AI for plan only.
4. Review plan.
5. Ask AI for tests.
6. Run tests and confirm failure.
7. Ask AI for implementation.
8. Run checks.
9. Review diff.
10. Commit.
11. Update docs.
```

## Chapter job

Make the book actionable for someone opening their editor tomorrow morning.

## Main beats

- Pick a bounded issue.
- Make acceptance criteria visible.
- Separate planning from patching.
- Prefer tests before implementation.
- Review the diff, not the confidence.
- Commit small.
- Leave a handoff.
