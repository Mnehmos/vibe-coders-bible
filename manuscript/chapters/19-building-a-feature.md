# Chapter 19 - Building A Feature

Part: V - AI-Assisted Development Workflows

## Thesis

A feature is safest when the issue, acceptance criteria, test plan, implementation, and PR all describe the same behavior.

## Key line

Do not ask the agent to build a feature until the feature has a shape.

## Chapter job

Turn feature work into a repeatable AI-assisted loop.

## Main beats

- Issue.
- Acceptance criteria.
- Test plan.
- Implementation plan.
- Agent prompt.
- Diff review.
- Test run.
- PR.

## Practical artifacts

- `FEATURE_BRIEF.md`
- `feature-agent-prompt.md`
- Feature issue template.

## Feature loop

```text
Issue -> acceptance criteria -> test plan -> proposal -> tests -> implementation -> validation -> PR
```

## Draft notes

Stress that feature scope should be visible before generation starts.
