# Chapter 18 - Starting A Project

Part: V - AI-Assisted Development Workflows

## Thesis

A project started with AI still needs a product brief, architecture sketch, validation path, and memory layer from day one.

## Key line

The first scaffold should include the controls, not just the app.

## Chapter job

Give builders a disciplined starting workflow that does not smother speed.

## Main beats

- Product brief.
- User stories.
- Architecture sketch.
- Tech stack choice.
- File structure.
- First tests.
- CI from day one.
- README from day one.
- Early agent instructions.

## Practical artifact

Deliverable: `templates/ai-project-brief.md`

Add a starter flow:

1. Define the user and job.
2. Pick the smallest architecture that can survive validation.
3. Create the repo memory files.
4. Add the first test command.
5. Make the first commit boring and reviewable.

## Draft notes

This chapter should help a new builder start correctly without requiring enterprise process.
