# Chapter 22 - Documentation As A Build Artifact

Part: V - AI-Assisted Development Workflows

## Thesis

Documentation should be generated, reviewed, and updated as part of the build discipline, not treated as a decorative afterthought.

## Key line

Documentation is not what you write after building. Documentation is how the next builder survives the build.

## Chapter job

Show how AI makes documentation cheaper while making verification of documentation more important.

## Main beats

- Docs from code.
- Docs from tests.
- Docs from issues.
- Docs from changelogs.
- Agent-readable docs.
- Human-readable docs.
- Avoiding stale docs.
- Docs as repo memory.

## Practical artifact

Add a "docs from diff" prompt and checklist:

- What behavior changed?
- What setup changed?
- What commands changed?
- What examples changed?
- What claims need source verification?

## Draft notes

Warn that AI can generate fluent false docs. Documentation claims need the same commit boundary as code claims.
