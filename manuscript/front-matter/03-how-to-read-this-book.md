# How To Read This Book

This book is designed for multiple paths.

## Beginner path

Read the doctrine, workflows, checklists, and examples. Focus on the practical loops:

- Propose.
- Validate.
- Commit.
- Trace.

Copy the templates before trying to invent your own process.

## Professional path

Read the controls, architecture, case studies, and templates. Focus on where attention is currently carrying work that a system should enforce.

## Builder path

Use the repo as a starter kit. Copy the templates into an active project, create one issue, and run the daily workflow on a real change.

## Audio path

The audiobook should preserve the field-manual rhythm. Tables may need narrated versions, and code blocks should be summarized unless they are essential to the point.
