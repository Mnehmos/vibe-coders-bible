# Appendix A - Glossary

## Vibe coding

Building software by steering AI-generated work through natural language, examples, feedback, and iteration.

## Proposal

An AI-generated draft, plan, diff, command, schema, or artifact that has not yet crossed a commit boundary.

## Commit

Any point where output becomes accepted state: Git history, database state, deployed code, published content, user-visible behavior, or trusted documentation.

## Validator

A deterministic or reviewable mechanism that checks a proposal before commit.

## Reflex

A validation step that fires automatically before humans have to remember it.

## Schema

A typed boundary that defines what shape of data or action is valid.

## State substrate

The durable structured state from which rendered output, narration, UI, media, or explanation is derived.

## Trace

The recorded path of intent, proposal, validation, commit, and result.

## Source of truth

The artifact or system that owns reality for a specific claim.

## Mixed model output

An LLM response that contains multiple output types, such as prose, code, commands, claims, citations, and tool calls.

## Trust boundary

The line between untrusted proposal and accepted state.

## Hierarchy of controls

A risk-reduction framework that prefers eliminating hazards and engineering controls before relying on process, prompts, or attention.
